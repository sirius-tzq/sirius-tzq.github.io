分组查询

语法 说明
GOUP BY label 按照label分组，每组的内容将由聚合函数来求。注意如果要混用WHERE查询的话，分组查询要在条件查询之后

代码的运算流程大致如下：先列出了department_id和employee_id两列，筛选出salary＞5000的。在新表中将department_id相同的作为一组，对该组的employee_id作COUNT运算(也就是人员数目)。这样就达成了计算出每部门人员数的目的。

需要注意的是，分组查询的SELECT列只能是GROUP BY查询列和聚合函数列。不能有其他列。

  • 分组过滤查询

对于已经分组查询后的表，进一步筛选。注意不能使用WHERE筛选，因为那是在分组前筛选。而应改用HAVING筛选。

在上一步分组的条件下，进一步筛选出平均工资高于8000的部门和其下分工部门。

查询总结

  1. FROM：找到数据源列出所有数据
  2. WHERE：按列的标准第一次筛选行，还没有开始筛选列
  3. GROUP BY：按列的标准将相同行归为同一组
  4. HAVING：按列的标准筛选出组。至此完成了行的筛选。
  5. SELECT：在筛选完的表中抽出相应的列。
  6. ORDER BY：按列的标准将组进行排序。
  7. LIMIT：抽出部分行出来。

首先只考虑薪资大于5000的员工，这些员工先以管理者分组，再细分到各管理者下的各部门，统计出各部门的平均薪资和人数。筛选完毕后再筛选出部门人数不少于2人的部门。

这些最后筛选出的部门竞选出平均薪资最高的前十部门和对应的管理员。

约束

表中的一行数据代表一个实体，实体是唯一的。即不能存在两个完全一样的行。

  • 主键约束 PRIMARY KEY
  • 唯一约束 UNIQUE

UNIQUE表示该列不允许出现重复项，可以为NULL。

  • 自动增长列 AUTO_INCREMENT

AUTO_INCREMENT必须跟随着PRIMARY KEY，从第一个添加的实体开始PRIMARY KEY初设为1，每添加一项PRIMARY KEY自增。这也意味着使用INSERT添加时可以省略PRIMARY KEY项。

  • 非空约束 NOT NULL

NOT NULL要求该列不能出现空值，常与UNIQUE连用。

  • 默认值约束 default
